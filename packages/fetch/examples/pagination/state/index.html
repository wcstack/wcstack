Pagination — wcstack / state

Declarative <wcs-fetch> + data-wcs — no JS glue

→ →

Pure HTML wiring: a computed itemsFetch.url drives one <wcs-fetch>, and data-wcs paths render the result — no fetch(), no loading/abort glue. Changing page rebuilds the URL, so <wcs-fetch> refetches and aborts the previous request automatically. It hits the shared /api/items server, the same endpoint the React / Vue / signals / Vanilla demos use.
