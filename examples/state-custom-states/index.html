wcstack: :state() showcase

Every wcstack I/O node reflects its boolean output (loading, error, connected, …) onto its own element via CustomStateSet, so plain CSS can select it with :state(loading) — no JavaScript state-subscription code required. The spinner, error banners, and connection dot on this page are driven entirely by the <style id="state-css"> block below; open View Source or expand "Show the CSS" at the bottom of the page.

Requires @wcstack/fetch / @wcstack/websocket v1.17.0 or later. Older CDN-resolved versions still work — the components just never set their :state(), so the selectors above silently never match.

⚠ One of the demo nodes below reported an error. Its local banner (in its own card) has the details.

2. Connection indicator — <wcs-ws>

The dot below turns green the instant the WebSocket opens, amber while connecting/reconnecting, and the card gets a red outline on error — again, entirely from :state(connected) / :state(loading) / :state(error) on #ws-conn.

This dot's color is not bound to any state path — it is pure CSS reading <wcs-ws>'s own :state().

Inspecting the states

:state() is invisible in DevTools' Elements panel — there is no attribute to see. Every wcstack Shell offers an opt-in debug-states attribute that mirrors state changes onto data-wcs-state-* attributes purely for inspection (never used for styling — write your CSS against :state(), as above).
