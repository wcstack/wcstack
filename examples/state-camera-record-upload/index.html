wcstack: camera → record → upload

The whole pipeline is one straight line and the live MediaStream never touches state. <wcs-camera> acquires it and shows the preview; on stream-ready the raw handle is handed to <wcs-recorder> as a transient command-token argument ($command.feedRecorder.emit(stream)) — it passes through the token bus and is never stored as a reactive value. Only derived values (permission, recording flag, the recorded Blob, an object URL) live in state. The recorded clip becomes a File and flows into the existing <wcs-upload> node.
