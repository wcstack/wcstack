wcstack: pomodoro timer

state + timer + wakelock + notification — the screen stays awake only while you focus, and the session-end notification is the "start next" button.

completed:
  • wakelock: the lock is requested only while a focus session is running — pause (or a break) releases it. Watch the "screen" pill; held is the OS truth, not the request.
  • timer: <wcs-timer> is just a declarative 1s metronome (tick → event token); the pomodoro state machine is ~20 lines of state.
  • notification: the session-end notification carries data.next and its click flows back through eventToken.clicked — clicking it starts the next session, even from another tab.
  • Pick the 6 sec / 3 sec (demo) durations to watch a full focus → break → focus cycle without waiting 25 minutes.
  • Serve over http://localhost or HTTPS — wake lock and notifications need a secure context.
