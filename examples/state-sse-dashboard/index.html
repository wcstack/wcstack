  • Same job, two owners: <wcs-sse> and $streams compete for the same work, so this page runs them side by side instead of chaining them. Rule of thumb: wiring events at elements → the tag; folding/switching inside state → $streams.
  • The host switch is the punchline: the left panel's reset is three manual lines in setHost(); the right panel resets itself because args reads host — the dependency-driven restart aborts the old EventSource and starts from initial.
  • Named events: events="metric,deploy" funnels every named SSE event into the single message output — message.event tells you which one fired.
  • network is a pure monitor of the Network Information API (connection quality, not online/offline) — no attributes, no commands, five outputs. It is Chromium-only and the tile says so elsewhere.
  • Both folds are bounded (last-20 window): backpressure is explicitly abandoned, so unbounded accumulation on a long-lived stream is the one mistake this page must not make.
